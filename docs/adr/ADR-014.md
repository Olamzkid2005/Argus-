# ADR-014: Configuration Hierarchy

**Status:** Accepted

**Date:** 2026-06-02

## Context

Argus needs to support configuration from multiple sources (CLI flags, environment variables, project config, user config, built-in defaults) with a clear precedence order. The merge behavior for objects and arrays must be predictable to avoid surprising behavior when multiple sources define the same key.

## Decision

Establish a five-layer configuration hierarchy with strict precedence:
1. CLI flags (highest priority)
2. Environment variables
3. Project config (`./argus.config.yaml`)
4. User config (`~/.argus/config.yaml`)
5. Built-in defaults (lowest priority)

Objects are deep-merged across layers — nested keys from higher-priority sources override individual keys without discarding unrelated keys from lower-priority sources. Arrays use replace semantics — a higher-priority array completely replaces a lower-priority one (never merges). The `ResolvedValue<T>` wrapper tracks the origin layer of every value for debugging. Workflow YAML files are explicitly excluded from this hierarchy — they define the assessment plan, not runtime configuration.

## Consequences

- Predictable override behavior — CLI flags always win, arrays never produce surprising combinations
- Deep-merge for objects allows partial overrides (e.g., overriding `db.host` without re-specifying `db.port`)
- `ResolvedValue<T>` origin tracking aids debugging — users can see which layer provided each setting
- Replace semantics for arrays may surprise users who expect array merging — documented behavior mitigates this
- Workflow YAML exclusion keeps the plan separate from configuration, avoiding ambiguity about whether a value is a plan instruction or a runtime setting

## Alternatives Considered

- **Flat config (no layering):** Requires users to specify every value in a single file, with no ability to set global defaults or override per project. Inflexible for diverse usage patterns.
- **Single config file (no per-project override):** Forces all projects on a machine to share the same configuration, which is impractical when different assessments need different tool paths or credential locations.
- **Merge-all-arrays (concatenation):** Produces unpredictable behavior — a user who sets `exclude_domains` in their user config would silently append to the project list rather than replacing it, potentially causing unexpected exclusions.
